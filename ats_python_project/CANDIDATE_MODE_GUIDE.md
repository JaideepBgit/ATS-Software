## What is Candidate Mode?

Candidate Mode is designed for **YOU** - the job seeker who wants to improve their resume. Instead of evaluating candidates, the AI becomes your personal resume coach, helping you understand:

- **Why** certain aspects were flagged as concerns
- **How** to rewrite specific resume points to better match the job
- **What** changes would increase your match score
- **Which** skills or experiences to emphasize

## How to Use It

### 3. Place Your Resume

Put the job description in: `data/job_description.txt`

## Real Example Conversation

**You:** Why was my experience flagged as a concern?

**AI:** Your experience shows 3 years total, but the job requires 5+ years. Additionally, your experience is described in generic terms without specific technologies or measurable outcomes. For example, you wrote "Developed web applications" but didn't mention which frameworks, languages, or the scale/impact of your work.

---

**You:** How can I rewrite my first work experience to be better?

**AI:** Let me show you a before/after:

**BEFORE:**
```
Software Developer at ABC Corp (2021-2023)
‚Ä¢ Developed web applications
‚Ä¢ Worked with team members
‚Ä¢ Fixed bugs and improved code
```

**AFTER:**
```
Full-Stack Software Developer at ABC Corp (2021-2023)
‚Ä¢ Architected and deployed 5 Python/Django web applications serving 50K+ daily users, reducing page load time by 40%
‚Ä¢ Collaborated with cross-functional team of 8 developers using Agile methodology to deliver features 25% faster
‚Ä¢ Resolved 200+ production bugs and refactored legacy codebase, improving code coverage from 45% to 85%
```

**Why this is better:**
- Specific technologies (Python, Django)
- Quantified impact (50K users, 40% improvement)
- Shows scale and responsibility
- Uses strong action verbs
- Demonstrates technical depth

---

**You:** What keywords am I missing?

**AI:** Based on the job description, you're missing these critical keywords:
- **Cloud platforms:** AWS, Docker, Kubernetes
- **CI/CD:** Jenkins, GitHub Actions
- **Testing:** pytest, unit testing, TDD
- **Databases:** PostgreSQL, Redis
- **API design:** REST, GraphQL

Add these where relevant in your experience. For example, if you used AWS but didn't mention it, add "Deployed applications to AWS EC2 with Docker containerization."

---

**You:** How can I get to 90%+ match?

**AI:** Here are the top 3 changes to boost your score:

1. **Add Cloud Experience (Impact: +15%)**
   - If you've used AWS/Azure/GCP, add specific examples
   - If not, get AWS certification and add it
   - Mention any containerization (Docker) or orchestration (Kubernetes)

2. **Quantify Everything (Impact: +10%)**
   - Add metrics to every bullet point
   - Show scale: users, data volume, performance improvements
   - Demonstrate business impact: revenue, cost savings, efficiency

3. **Emphasize Leadership (Impact: +8%)**
   - The job wants team leadership
   - Reframe your collaboration as "Led team of X developers"
   - Add mentoring, code reviews, technical decisions you made

Make these changes and your score should jump to 88-92%.

## Tips for Best Results

### ‚úÖ DO:
- Be specific about which part of your resume you want to improve
- Ask for concrete before/after examples
- Request prioritized action items
- Ask about specific technologies or skills
- Get clarification on why something is a weakness

### ‚ùå DON'T:
- Ask vague questions like "How do I improve?"
- Expect the AI to rewrite your entire resume at once
- Ignore the specific job requirements
- Add false information to your resume
